# OpenCode Remote Channel Plugin

## 项目概述

**目标**: 通过微信远程控制 OpenCode，实现**持续的多轮对话**，保持上下文连贯。

**核心原则**: 
- **一个会话，多次交互** — 同一个对话窗口里可以持续聊天
- **上下文保持** — AI 能记住之前的对话内容
- **避免碎片化** — 不是每次消息都变成单独会话

**参考**: 基于 `@tencent-weixin/openclaw-weixin` 插件架构

## 对话流程设计

### 正常对话（默认行为）
```
用户: 你好
AI: 你好！有什么可以帮你？
用户: 帮我写一个 hello world
AI: [代码]
用户: 帮我改成彩色版本
AI: [修改后的代码]
用户: 再加个背景图
AI: [继续修改]
...
(继续在同一会话中，对话上下文一直保持)
```

### 主动切换会话
```
用户: /sessions      → 列出所有会话
用户: /switch abc    → 切换到 abc 会话
(切换后继续对话)
```

## 消息命令设计

### 对话命令（最重要）
| 命令 | 说明 | 示例 |
|------|------|------|
| **无前缀消息** | 直接对话，保持当前 session | `帮我写代码` |
| `/chat <消息>` | 明确对话意图 | `/chat 帮我写代码` |
| 直接发送图片 | 上传图片继续讨论 | [图片] |

### 会话管理
| 命令 | 说明 | 示例 |
|------|------|------|
| `/sessions` | 列出所有会话 | `/sessions` |
| `/switch <session>` | 切换会话 | `/switch abc123` |
| `/new` | 创建新会话 | `/new` |
| `/delete <session>` | 删除会话 | `/delete abc123` |
| `/clear` | 清除当前会话上下文 | `/clear` |

### Agent/模型管理
| 命令 | 说明 | 示例 |
|------|------|------|
| `/agents` | 列出 Agent | `/agents` |
| `/agent <name>` | 切换 Agent | `/agent opencode` |
| `/models` | 列出模型 | `/models` |
| `/model <name>` | 切换模型 | `/model glm-5` |

### 目录/系统
| 命令 | 说明 | 示例 |
|------|------|------|
| `/pwd` | 当前目录 | `/pwd` |
| `/cd <path>` | 切换目录 | `/cd ~/project` |
| `/ls` | 列出目录 | `/ls` |
| `/exec <cmd>` | 执行命令 | `/exec ls -la` |
| `/help` | 帮助 | `/help` |

## 技术要点

### 1. Session 绑定
- 用户首次对话时创建 session 并绑定
- 后续消息使用相同 session
- session ID 存储在用户侧或服务端

### 2. 上下文保持
- OpenCode 的 session 管理机制
- 消息历史累积
- 不主动刷新/重建会话

### 3. 微信消息处理
- 文本消息 → 对话
- 图片消息 → 上传后继续对话
- 命令消息 → 解析执行
